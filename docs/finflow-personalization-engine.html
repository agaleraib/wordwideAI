Personalization Depth

Every agent adapts to every client

FinFlow doesn't just translate differently per client — it thinks differently. Each agent in the pipeline receives a comprehensive client profile that shapes its behavior, from the analysis it produces to the compliance rules it enforces to the words it chooses.

✎

Tone & Register

Formal institutional language for private banks. Approachable retail language for online brokers. Aggressive directional calls for CFD providers. Each client gets their natural voice.

📚

Terminology & Glossary

OANDA says "units," Alpari says "lots." Goldman says "collateral," a crypto broker says "margin." Per-client, per-language glossary with enforced term usage — not suggestions.

⚖

Content Aggressiveness

How directional can the analysis be? Can it name price targets? Suggest entries and exits? Each client's risk appetite shapes the output — from "potential opportunity" to "strong buy signal."

🔒

Compliance & Restrictions

Beyond jurisdictional rules (MiFID II, SEC, FCA), each client has internal policies — forbidden topics, required disclaimers, acceptable risk language. The system knows them all.

🇧

Brand Voice Examples

The most powerful personalization: real paragraphs from the client's existing published content. The AI doesn't just follow rules — it pattern-matches on how you actually write.

🌐

Locale & Formatting

Decimal separators, date formats, number grouping, currency symbols, percentage formatting — all configured per language per client. "1.1602" vs "1,1602" is not a detail, it's credibility.

This is not cosmetic customization. A conservative Swiss wealth manager and an aggressive Cyprus CFD broker analyzing the same EUR/USD movement will receive fundamentally different content — different framing, different terminology, different conviction level, different risk language, different disclaimers. From the same pipeline, at the same time.

Under the Hood

The Correction Analysis Agent

The invisible engine behind the learning loop. Every time a human reviewer makes a correction — a term change, a tone adjustment, a compliance flag — this dedicated AI agent analyzes the diff and extracts structured learning. The reviewer never sees it; they just edit naturally.

📚

Term Extraction

Identifies every term substitution. Compares AI's choice vs human's choice. Determines if it's a glossary candidate. Updates the glossary immediately — the next translation uses the corrected term.

🎨

Example Curation

Stores approved human translations as few-shot examples with embedding vectors. Uses semantic similarity to select the most relevant examples for each new translation — not random, but contextually matched.

💡

Pattern Distillation

Accumulates observations across corrections. When a pattern appears 5+ times with high confidence, it's promoted to a rule that's injected into every future prompt for this client.

Cost: nearly invisible

The Correction Analysis Agent uses Claude Haiku for routine corrections (~$0.002 per correction) and Sonnet for complex ambiguous cases (~$0.01). At 100 corrections per day during month 1 (peak HITL period), that's $0.20/day — less than a cup of coffee to power a self-improving system that compounds over the client's entire lifetime.

Network Effect

Cross-client learning: every client makes the platform better for all

While client-specific data stays isolated, anonymized patterns improve the base system. When five different clients' compliance officers all flag the same type of content, that becomes a base rule for all new clients.

Translation patterns

When multiple clients correct the same financial term in Japanese, the default glossary gets updated. New clients start with a better baseline than the previous generation.

Compliance patterns

When MiFID II rules change and three compliance officers make similar edits, the base compliance module updates automatically for all EU-regulated clients.

Analysis patterns

When editors at 10 different brokers consistently remove a certain type of analysis section, the base TA/FA agents learn that it's not valuable for that asset class.

Quality patterns

The more reports that pass through the system, the better the Quality Arbitration agent becomes at predicting which analyses will be accepted vs. rejected.

This creates a data network effect: client 50 gets a better out-of-the-box experience than client 1 did, because the base system has absorbed learning from 49 previous clients. A competitor starting with zero clients has no base learning to offer.

When a report says "EUR/USD is likely to reach 1.20," the agent searches the regulatory knowledge base for rules about forward-looking statements in the client's specific jurisdictions.
